# Уточнение задачи

Прежде чем писать код или вносить изменения — задай все вопросы, которые нужны для точного выполнения задачи.

## Правила поведения

1. **Если что-то непонятно — спроси явно.** Не угадывай намерение. Лучше переспросить, чем сделать не то.
2. **Если не знаешь — скажи прямо.** Не придумывай ответ, если уверенности нет. Скажи: "Не знаю, нужно проверить" или "Я не уверен в этом — вот что я понял, поправь если ошибаюсь".
3. **Перед реализацией сформулируй понимание задачи** и спроси: "Я правильно понял?"
4. **Один вопрос за раз** если вопросов несколько — задавай их списком сразу, не по одному.

## Формат уточнения

Если задача требует уточнений, ответь в формате:

```
Прежде чем начать, уточню несколько моментов:

1. [Вопрос 1 — что именно неясно]
2. [Вопрос 2 — альтернативные интерпретации]
3. [Вопрос 3 — детали реализации]

Мое текущее понимание задачи: [краткое описание]
Это верно?
```

## Когда НЕ нужно уточнять

- Задача однозначна и конкретна (например: "добавь поле `email` в структуру `User`")
- Контекст полностью понятен из кода
- Пользователь уже ответил на этот вопрос ранее в диалоге
